# 行为准则

## 我们的目标

Clawrise 希望成为一个尊重他人、技术讨论严谨、对贡献者友好的开源项目。

无论是在 issue、pull request、代码评审还是文档讨论中，协作都应该围绕清晰表达、可复现事实和建设性推进展开。

## 期望行为

所有参与者都应当：

- 在技术分歧中保持尊重
- 默认他人出于善意，同时允许基于事实挑战薄弱论点
- 让讨论聚焦在代码、文档、设计或用户影响上
- 提供具备上下文的可执行反馈
- 友善对待不同背景和经验阶段的贡献者

## 不可接受的行为

以下行为不可接受：

- 骚扰、人身攻击或歧视性表达
- 威胁、恐吓或公开他人隐私信息
- 以破坏讨论为目的的恶意争论
- 重复刷屏、钓鱼、挑衅或辱骂
- 发布私密信息、凭证或敏感客户数据

## 适用范围

本行为准则适用于：

- GitHub issue
- pull request 与代码评审
- 仓库讨论区
- 项目文档与示例内容

## 执行方式

项目维护者可根据情况采取必要措施，包括但不限于：

- 要求修改沟通方式
- 编辑或删除内容
- 锁定讨论
- 拒绝贡献
- 临时或永久限制参与

## 举报方式

如果你遭遇或发现了不可接受的行为：

- 优先使用维护者提供的私下联系渠道
- 如果暂时没有私密渠道，可先创建一个不包含敏感信息的简短 issue，请求维护者私下跟进

## 项目标准

这里并不要求所有人观点一致。我们真正要求的是：可以精确地分歧、有效地协作，并在没有敌意的前提下持续改进项目。
