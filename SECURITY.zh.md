# 安全策略

## 支持范围

Clawrise 仍处于 1.0 之前的快速演进阶段。

当前阶段仅承诺为默认分支上的最新代码，以及最近一次发布版本（如果存在）提供安全修复支持。

| 版本 | 是否支持 |
| --- | --- |
| `main` / 默认分支 | 是 |
| 更早分支或历史快照 | 否 |

## 如何报告漏洞

在维护者完成调查和响应之前，请不要公开披露安全漏洞细节。

推荐的报告路径：

1. 如果仓库启用了 GitHub 私密漏洞报告功能，请优先使用它。
2. 如果没有私密报告入口，请创建一个标题为 `[security] private coordination requested` 的公开 issue，但不要写入漏洞利用细节。
3. 在该 issue 中只保留建立私下沟通渠道所需的最少信息。

报告漏洞时，建议提供：

- 受影响的命令、包或 operation
- 已知受影响版本或提交哈希
- 严重程度与潜在影响
- 清晰的复现步骤
- 是否可能泄露凭证、租户数据或令牌
- 如有的话，附上建议缓解方案

## 响应预期

维护者会尽量做到：

- 在 7 天内确认收到报告
- 复现并评估问题影响
- 协调修复方案与发布时间
- 在适合公开披露时感谢报告者

## 敏感信息处理

- 不要在 issue 或 pull request 中提交真实凭证。
- 分享请求数据、日志和租户标识前请先脱敏。
- 如果你不确定某项信息是否敏感，请按敏感信息处理。
