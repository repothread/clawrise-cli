# Support

If you need help with Clawrise, use the path below that best matches your problem.

For the Chinese version, see [SUPPORT.zh.md](SUPPORT.zh.md).

## Where to Go

- Bug reports: open a GitHub issue with reproduction steps, expected behavior, actual behavior, and sanitized logs.
- Feature requests: open a GitHub issue describing the workflow gap, user impact, and alternatives considered.
- Security issues: follow [SECURITY.md](SECURITY.md) and avoid posting exploit details publicly.
- Contribution questions: read [CONTRIBUTING.md](CONTRIBUTING.md) before opening large implementation changes.
- Project direction questions: check the public roadmap in [docs/en/roadmap.md](docs/en/roadmap.md) first.

## Information That Helps Maintainers

Please include:

- OS and Go version
- Clawrise version or commit hash
- affected command, plugin, or operation name
- exact command you ran
- sanitized config snippets or payload samples when relevant
- what you expected, what happened instead, and how to reproduce it

## What Maintainers Can Usually Help With

- build and test failures in the repository
- config bootstrap and `doctor` output interpretation
- plugin installation and verification problems
- operation discovery via `spec list`, `spec get`, and `spec status`
- documentation gaps and confusing onboarding flows

## What Usually Slows Down Support

- screenshots without copyable command output
- missing reproduction steps
- issues reported without the provider, profile, or operation involved
- logs that hide the failing command entirely
- reports containing real secrets that then need to be rotated

## Response Expectations

This is an open source project and response times are best-effort.

If you are proposing a major change, check the roadmap first and open an issue before spending time on a large pull request.
