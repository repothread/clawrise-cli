# 获取支持

如果你在使用 Clawrise 时遇到问题，请优先根据问题类型选择合适的沟通入口。

英文版本见 [SUPPORT.md](SUPPORT.md)。

## 应该去哪里

- 缺陷问题：创建 GitHub issue，并提供复现步骤、预期行为、实际行为和脱敏日志。
- 功能建议：创建 GitHub issue，说明当前工作流缺口、用户价值和考虑过的替代方案。
- 安全问题：遵循 [SECURITY.zh.md](SECURITY.zh.md) 中的流程，不要公开发布漏洞利用细节。
- 贡献相关问题：在发起较大实现改动前先阅读 [CONTRIBUTING.zh.md](CONTRIBUTING.zh.md)。
- 项目方向问题：先查看公开路线图 [docs/zh/roadmap.md](docs/zh/roadmap.md)。

## 哪些信息能帮助维护者更快定位问题

建议尽量提供：

- 操作系统与 Go 版本
- Clawrise 版本或提交哈希
- 受影响的命令、plugin 或 operation 名称
- 你实际执行的命令
- 必要的脱敏配置片段或输入样例
- 你的预期结果、实际结果以及复现步骤

## 维护者通常可以帮助处理的问题

- 仓库内的构建与测试失败
- 配置初始化和 `doctor` 输出解读
- plugin 安装与校验问题
- `spec list`、`spec get`、`spec status` 等发现能力相关问题
- 文档缺失或接入流程不清晰的问题

## 哪些情况通常会拖慢支持效率

- 只有截图，没有可复制的命令输出
- 缺少复现步骤
- 没写清楚 provider、account 或 operation
- 日志里看不出失败命令
- 提交了真实密钥，导致必须先做密钥轮换处理

## 响应预期

这是一个开源项目，维护者的响应时间以实际精力为准，并非服务级承诺。

如果你准备推动一个较大的改动，建议先查看路线图，再先开 issue 讨论，避免投入后发现方向不一致。
