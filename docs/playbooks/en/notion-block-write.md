# Write Notion Blocks Safely

Operations:

- `notion.page.create`
- `notion.block.append`
- `notion.block.update`
- `notion.block.get`
- `notion.page.markdown.get`

## Read before write

```bash
clawrise notion.block.get --json '{"block_id":"block_demo"}'
clawrise notion.page.markdown.get --json '{"page_id":"page_demo"}'
```

## Inspect the contract first

```bash
clawrise spec get notion.block.append
clawrise spec get notion.block.update
```

## Supported block input shapes

Clawrise supports both of these block payload styles:

- shorthand top-level fields such as `text`, `rich_text`, `children`, and `checked`
- provider-native nested block bodies such as `paragraph.rich_text` and `to_do.checked`

When both shapes are present on the same block, the top-level fields take precedence.

## Dry-run with shorthand fields

```bash
clawrise notion.block.append --dry-run --json '{
  "block_id":"page_demo",
  "children":[
    {"type":"heading_1","text":"Weekly Notes"},
    {"type":"paragraph","text":"Generated by Clawrise."},
    {"type":"to_do","text":"Ship the fix","checked":false}
  ]
}'
```

## Dry-run with provider-native nested block bodies

```bash
clawrise notion.block.append --dry-run --json '{
  "block_id":"page_demo",
  "children":[
    {
      "type":"heading_1",
      "heading_1":{
        "rich_text":[
          {"type":"text","text":{"content":"Weekly Notes"}}
        ]
      }
    },
    {
      "type":"paragraph",
      "paragraph":{
        "rich_text":[
          {"type":"text","text":{"content":"Generated by Clawrise."}}
        ]
      }
    },
    {
      "type":"to_do",
      "to_do":{
        "checked":false,
        "rich_text":[
          {"type":"text","text":{"content":"Ship the fix"}}
        ]
      }
    }
  ]
}'
```

## Update one block with a provider-native body

```bash
clawrise notion.block.update --dry-run --json '{
  "block_id":"block_demo",
  "block":{
    "type":"paragraph",
    "paragraph":{
      "color":"green_background",
      "rich_text":[
        {"type":"text","text":{"content":"Updated by Clawrise."}}
      ]
    }
  }
}'
```

## Verify after the real write

```bash
clawrise notion.block.get --json '{"block_id":"block_demo"}'
clawrise notion.page.markdown.get --json '{"page_id":"page_demo"}'
```

## Practical guidance

- keep `--dry-run` in the loop until the payload shape is stable
- if you are reusing block objects from another Notion read or transform step, keeping the provider-native nested body is safe
- prefer `notion.page.markdown.update` when the task is really markdown-first page editing instead of structured block editing
