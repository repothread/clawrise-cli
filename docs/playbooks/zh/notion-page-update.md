# 更新 Notion 页面

适用 operation：

- `notion.page.update`
- `notion.page.markdown.update`
- `notion.page.get`
- `notion.page.markdown.get`

如果要做结构化 block 写入，请改看：

- `docs/playbooks/zh/notion-block-write.md`

适用场景：

- 更新页面标题或属性
- 用 markdown 命令替换页面正文

## 1. 先读取页面

```bash
clawrise notion.page.get --json '{"page_id":"page_demo"}'
```

```bash
clawrise notion.page.markdown.get --json '{"page_id":"page_demo"}'
```

## 2. 更新页面属性或标题

```bash
clawrise notion.page.update --dry-run --json '{
  "page_id":"page_demo",
  "title":"Updated project notes"
}'
```

## 3. 对属性更新做验证或查看上游载荷

```bash
clawrise notion.page.update --debug-provider-payload --verify --json '{
  "page_id":"page_demo",
  "title":"Updated project notes"
}'
```

## 4. 归档或移入回收站

```bash
clawrise notion.page.update --dry-run --json '{
  "page_id":"page_demo",
  "in_trash":true
}'
```

当前 Notion API 原生字段是 `in_trash`，`archived` 仍作为向后兼容别名保留。

## 5. 用 markdown 替换正文

```bash
clawrise notion.page.markdown.update --dry-run --json '{
  "page_id":"page_demo",
  "type":"replace_content",
  "replace_content":{
    "new_str":"# Updated title\n\nGenerated by Clawrise."
  }
}'
```

对于 Notion enhanced markdown：

- 句子或列表项里的行内提及要用 `<mention-page>` 和 `<mention-database>`
- `<page>` 和 `<database>` 只应用作独立 block 引用，必须单独占一行

真正写入时移除 `--dry-run` 即可。

## 验证建议

- 先运行 `clawrise spec get notion.page.markdown.update`
- 先读后写，避免覆盖掉页面中已有内容
- 若只改标题或属性，优先使用 `notion.page.update`
- 需要写后确认结果时，在 `notion.page.update` 上加 `--verify`
- 需要查看最终上游请求和响应时，在 `notion.page.update` 上加 `--debug-provider-payload`
