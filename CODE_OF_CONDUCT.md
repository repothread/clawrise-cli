# Code of Conduct

## Our Goal

Clawrise aims to be a respectful, technically rigorous, and welcoming open source project.

Participation in this repository, its issues, discussions, and pull requests should help move ideas forward through clear reasoning, reproducible evidence, and constructive collaboration.

## Expected Behavior

Participants are expected to:

- be respectful in technical disagreement
- assume good intent while still challenging weak reasoning
- keep discussions focused on the code, docs, design, or user impact
- share actionable feedback with enough context to be useful
- welcome contributors with different backgrounds and experience levels

## Unacceptable Behavior

The following behavior is not acceptable:

- harassment, personal attacks, or discriminatory language
- threats, intimidation, or doxxing
- bad-faith arguments intended to derail discussion
- repeated low-signal spam, trolling, or abusive behavior
- publishing private information, credentials, or sensitive customer data

## Scope

This code of conduct applies to:

- GitHub issues
- pull requests and reviews
- repository discussions
- project documentation and examples

## Enforcement

Project maintainers may take any action they consider appropriate, including:

- requesting a change in behavior
- editing or removing content
- locking conversations
- rejecting contributions
- temporarily or permanently restricting participation

## Reporting

If you experience or witness unacceptable behavior:

- use a private maintainer contact path when available
- otherwise open a minimal issue asking for private follow-up without posting sensitive details

## Project Standard

The goal is not perfect agreement. The goal is a repository where people can disagree precisely, collaborate effectively, and improve the project without hostility.
